# Votes

## Governor

The **Governor** system manages proposals and votes. Parameters and upgrades can be submitted to community vote.

## Proposal Types

Typical proposals include:

- **Game parameters** — Burn percentage, slot count, number range
- **Configuration** — Address updates (nums token, quote/DEX, etc.)
- **Treasury** — Fund allocation, spending, grants
- **Protocol upgrades** — Contract upgrades, new features

## Voting

- Voting power is typically based on **staked NUMS** in the vault
- Proposals have a submission period, voting period, and execution delay
- Quorum and approval thresholds are set in the Governor contract

## How to Participate

1. Stake NUMS in the vault
2. Review active proposals
3. Cast your vote
4. Proposals that pass move to timelock before execution
